# qemu-VIO 部署运行说明

## 1. 文件说明

部署运行需要以下两个文件：

```text
start_qemu_vio_master.sh      QEMU 启动脚本
vio_master.conf.example       配置文件模板
```

使用前复制一份配置文件：

```bash
cp vio_master.conf.example vio_master.conf
```

之后只需要修改 `vio_master.conf` 中的路径和参数。

## 2. 配置文件

打开配置文件：

```bash
vim vio_master.conf
```

需要重点确认以下配置：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `VIO_QEMU` | `/home/zyj/qemu-VIO/build/qemu-system-aarch64` | qemu-VIO 编译出的 QEMU 二进制路径 |
| `VM_IMAGE` | `/home/zyj/ubuntu-arm64/test-vdpa.qcow2` | Guest 虚拟机镜像路径 |
| `BIOS` | `/usr/share/edk2/aarch64/QEMU_EFI.fd` | AArch64 UEFI 固件路径 |
| `VHOSTDEV` | `/dev/vhost-vdpa-0` | vhost-vDPA 字符设备路径 |
| `VIO_VDPA_DEV` | `vdpa0` | vDPA stats 使用的设备名 |
| `VM_MEM` | `4096` | 主 VM 内存大小，单位 MB |
| `VM_SMP` | `4` | 主 VM vCPU 数量 |
| `TASKSET_CPUS` | `4,5,6,7` | QEMU 绑定的 host CPU 核 |
| `QUEUES` | `4` | virtio-net/vhost-vDPA 队列对数量 |
| `VM_MAC` | `52:54:00:cb:45:11` | Guest 网卡 MAC 地址 |
| `QEMU_LOG` | `./logs/qemu-vio.log` | QEMU `-D` 输出的 VIO 调试日志 |
| `CONSOLE_LOG` | `./logs/qemu-console.log` | QEMU 控制台输出日志 |
| `VIO_HIGH_IOPS` | `100000` | SNOOP 切到 PASSTHROUGH 的高水位阈值 |
| `VIO_LOW_IOPS` | `20000` | PASSTHROUGH 切回 SNOOP 的低水位阈值 |
| `VIO_WINDOW_NS` | `1000000000` | IOPS 统计窗口，单位 ns |
| `VIO_COOLDOWN_NS` | `3000000000` | 两次切换之间的冷却时间，单位 ns |
| `VIO_PASSTHROUGH_MIN_NS` | `10000000000` | 进入 PASSTHROUGH 后最小驻留时间，单位 ns |
| `VIO_PASSTHROUGH_WARMUP_WINDOWS` | `2` | PASSTHROUGH 前几个采样窗口只用于预热 |
| `VIO_LOW_WINDOWS` | `5` | 连续低负载窗口达到该数量后切回 SNOOP |
| `VIO_DEBUG_TOUCH` | `1` | 是否打开 snoop touch 调试统计 |

## 3. 启动 QEMU

配置完成后执行：

```bash
./start_qemu_vio_master.sh
```

如需指定其他配置文件：

```bash
./start_qemu_vio_master.sh --config ./vio_master.conf
```

也可以临时覆盖单个配置项：

```bash
VM_IMAGE=/path/to/guest.qcow2 ./start_qemu_vio_master.sh
```

脚本启动后会使用 vhost-vDPA 网卡，并启用 `x-svq=on`：

```text
-netdev vhost-vdpa,vhostdev=<VHOSTDEV>,id=n0,queues=<QUEUES>,x-svq=on
```

## 4. 查看日志

QEMU 运行日志：

```bash
tail -f ./logs/qemu-vio.log
```

控制台日志：

```bash
tail -f ./logs/qemu-console.log
```

检查 VIO 是否启动：

```bash
grep -E 'threshold=|switch SNOOP|switch PASSTHROUGH|switch queue state|applying switch' ./logs/qemu-vio.log
```

正常情况下可以看到类似日志：

```text
VIO-VDPA: threshold=1, svq_control=1, initial_mode=SNOOP
VIO-VDPA: switch SNOOP -> PASSTHROUGH
VIO-VDPA: applying switch, target=PASSTHROUGH
VIO-VDPA: switch queue state, target=PASSTHROUGH, enable_svq=0
VIO-VDPA: switch PASSTHROUGH -> SNOOP
VIO-VDPA: applying switch, target=SNOOP
VIO-VDPA: switch queue state, target=SNOOP, enable_svq=1
```

## 5. 异常检查

如果启动失败或网络异常，可以先检查：

```bash
grep -E 'ring restore failed|unable to start vhost net|failed to query hardware|feature negotiation not complete|Invalid address' \
  ./logs/qemu-vio.log ./logs/qemu-console.log
```

还可以确认设备和文件是否存在：

```bash
ls -l /dev/vhost-vdpa-0
ls -l /home/zyj/qemu-VIO/build/qemu-system-aarch64
ls -l /home/zyj/ubuntu-arm64/test-vdpa.qcow2
ls -l /usr/share/edk2/aarch64/QEMU_EFI.fd
```

## 6. 关闭 QEMU

在 QEMU 控制台中退出，或在 host 上结束 QEMU 进程：

```bash
sudo pkill -f qemu-system-aarch64
```

关闭后可以确认：

```bash
ps aux | grep qemu | grep -v grep
```
